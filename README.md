# 단순화에 집착하는 개발자

복잡한 걸 단순하게 만드는 과정이 재밌어서 개발을 합니다.
"왜 이렇게 동작하지?"라는 질문을 끝까지 파고드는 걸 좋아합니다.

---

## My Vision

**"복합적인 문제를 하나씩 쪼개어, 가장 단순한 해결책을 찾는 것"**

거대한 문제 앞에서도 당황하지 않고, 논리적인 단위로 분해해서
근본적인 해결책을 찾으려 합니다.

좋은 코드는 읽기 쉬운 코드라고 생각합니다.
기능이 동작하는 것만큼, 다음에 이 코드를 읽을 사람을 생각하며 작성합니다.

동작 원리를 이해하지 못한 채로 넘어가지 않으려고 합니다.
공식 문서와 소스코드를 직접 확인하는 것이 그 방법입니다.

한 달 뒤, 지금보다 조금 더 발전한 제 모습을 기대하며 코드를 작성합니다.
